### There are two ways DVC tracks data

> **Note:** Use `dvc add` only for existing datasets. Do **not** use it for data that is already listed under `outs` in `dvc.yaml`.

```bash
1. Pipeline-generated data ✅
outs:
  - data

Track it using: dvc repro


2. Existing/manual data ✅

Example:

data/customer.csv   # Downloaded from Kaggle

Track it using: dvc add data/customer.csv 

This creates:

data/customer.csv.dvc
```

```bash
Rule of thumb
Generated by a DVC pipeline (outs) → Use dvc repro.
Existing dataset not produced by a pipeline → Use dvc add.
```
